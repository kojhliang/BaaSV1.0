Create a new repository

git clone ssh://git@172.17.21.139:10022/liangminhong/BaaS.git
cd BaaS
touch README.md
git add README.md
git commit -m "add README"
git push -u origin master

Existing folder or Git repository

cd existing_folder
git init
git remote add origin ssh://git@172.17.21.139:10022/liangminhong/BaaS.git
git add .
git commit
git push -u origin master

设置长文件名支持：
 git config --global core.longpaths true


1. 在工作空间，右键，打开Git Bash
2. clone主分支的代码（即下载主分支代码的过程）
执行命令： git clone ssh://git@172.17.21.139:10022/liangminhong/BaaS.git
3. 进入工程目录
cd   BaaS
4. 切换到分支并下载代码
git checkout -b V2 origin/V2 
git pull 
git pull origin Backend
5.单纯的切换分支
git checkout Backend


git fetch origin Backend:Backend2 从远程仓库下载origin分支到本地temp分支
git diff Backend2 比较本地和远程的不同
git merge Backend2  合并temp到本地仓库

5.提交代码
git add .
git commit -m "add node_module: dockerode multer unzip decompress walk request thread-sleep mysql"
git push -u origin V2



切换分支：
第一次先下载相应的分支：git checkout -b V2
git checkout V2，再执行git push -u origin V2
如果要提交到master：git checkout master
再执行git push -u origin master


提交全部：
git add -A

更新：
git remote -v  查看现有分支

git fetch origin master:temp  从远程仓库下载origin分支到本地temp分支
git diff temp  比较本地和远程的不同
git merge temp  合并temp到本地仓库

git fetch origin Backend:Backend2  从远程仓库下载origin分支到本地temp分支
git diff Backend2  比较本地和远程的不同
git merge Backend2  合并temp到本地仓库
